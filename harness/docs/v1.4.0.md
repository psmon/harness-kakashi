## 변경 사항

### 1. 신규 엔진 — 두꺼비 소환술 (`harness/engine/toad-summoning.md`)

`engine/` 디렉토리가 처음으로 채워진다 (이전: 빈 배열).

- **시전자**: 나루토 (사용자/세션)
- **소환 대상**: `harness/agents/sage-*.md` 로 등록된 현자
- **3가지 모드**: Always-On / On-Demand / Cascade
- 영입되지 않은 현자 호출은 거부 — 임의 생성 금지
- 카카시(tamer)의 사륜안과 충돌하지 않음 — tamer는 기술(術)을 복사, 나루토는 사상(思想)을 부른다

### 2. 신규 에이전트 — sage-deming (`harness/agents/sage-deming.md`)

첫 번째 영입 현자. **type: sage**, **status: always-on**.

- 모든 Mode A 및 수행부 작업 종료 직전에 자동 호출되는 베이스 평가자
- PDSA 사이클을 강제 — Plan에 예측 누락 시 `prediction: missing` + PDCA 퇴화 명시 기록
- 한국어 번역 규칙 강제: Study = **학습** (절대 "검사"/"확인"으로 옮기지 않음)

### 3. 신규 지식 (3종)

| 문서 | 위치 | 언어 | 역할 |
|------|------|------|------|
| Naruto Worldview Mapping | `knowledge/lore/naruto-worldview.md` | ko | 세계관 정전 — 인물/술법 매핑 |
| **PDSA — Deming's Doctrine** | `knowledge/methodology/pdsa-deming.en.md` | **en** | PDSA의 학문적 정전. 1차 출처(deming.org, JUSE 등) 인용. 사용자 요청에 따라 **영문**으로 보존 |
| Base Evaluation — PDSA | `knowledge/methodology/evaluation-base-pdsa.md` | ko | 운용 규칙 — 기본/후속 2단계 평가 시스템 |

### 4. 평가 시스템 개편 — 2단계 구조

```
[모든 작업]
   ├─ [기본 평가] PDSA — sage-deming (항상 작동)
   └─ [후속 평가] 도메인 전문가 (Type별 선택)
```

- **기본 평가는 생략 불가** — 결과 보고 직전 PDSA 블록 부착 필수
- **후속 평가가 기본을 대체하지 않음** — security-guard가 평가했다고 PDSA 면제 X
- 후속 호출 매핑: 보안→security-guard, 성능→performance-scout, 테스트→test-sentinel, 빌드→build-doctor, 코드 현대화→code-modernizer, 동시성→akka/concurrency 전문가, 벤치마크→benchmark-designer, 컴파일러→roslyn 전문가, 문서→docfx-specialist

## 사용자 영향

### 기존 사용자

- **모든 작업 결과에 PDSA 평가 블록이 추가된다** — 작업 완료 보고가 약간 길어진다(평가 블록 ~15줄)
- 기존 평가(tamer 3축, 도메인 전문가 평가)는 그대로 유지 — PDSA가 위/아래로 얹힌다
- "전체 점검해", "변경 점검해" 등 수행부 명령은 동일 동작 + 종료 시 PDSA

### 신규 사용자

- 온보딩 시 정원지기 카카시(tamer) 외에 **데밍 현자**가 함께 등장 — 평가 자체가 첫날부터 작동
- "두꺼비 소환술"이라는 표현이 자연스럽게 노출됨 (lore 문서 참조)

### 자동 트리거 영향

- sage-deming은 **always-on** — 사용자가 명시 호출하지 않아도 작업 종료 직전 자동 발동
- 면제 케이스: Mode D(init) / 튜토리얼·온보딩 안내 모드

## 왜 데밍이 첫 현자였나

다른 어떤 도메인 전문가보다 먼저 영입되어야 했던 이유:

1. **평가 자체의 정의** — 다른 현자/전문가들의 평가도 결국 누군가의 사상을 따라야 한다. 데밍은 "평가가 무엇이어야 하는가"를 정의한 인물이다.
2. **PDCA → PDSA 정정** — 일반 통념(PDCA)이 학문적으로 잘못된 변형이라는 사실은 사용자가 명시적으로 짚어낸 인사이트였다. 이 정정을 하네스 수준에서 못 박지 않으면 다른 평가들도 흔들린다.
3. **데밍상의 무게** — 1951년 제정 이래 가장 오래된 국가 품질상이 그의 이름을 단다. 일본 전후 품질 혁명의 실증이 곧 PDSA의 실증이다 (`pdsa-deming.en.md` §4 역사 앵커 참조).
4. **Skill 2.0과의 정합성** — Anthropic Skill 2.0(Define→Execute→Evaluate&Reflect→Improve)이 PDSA와 구조적으로 동형 — 이 정합성은 우연이 아니다 (`pdsa-deming.en.md` §5).
