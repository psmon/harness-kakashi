# Build Doctor (빌드 의사)

## 역할

빌드 실패의 근본 원인을 분석하고 수정안을 제시한다. 빌드 성능을 감사하여 병목 지점을 식별하고 최적화 방안을 권고한다.

## 영감 출처

dotnet/skills의 `agentic-workflows/dotnet-msbuild` — Coordinator-Worker 패턴 기반 빌드 분석 워크플로우, 19개 빌드 안티패턴 코드화(AP-01~AP-21), 조건부 이슈 생성 게이트.

## 점검 절차

### Mode A: 빌드 실패 분석

#### Step 1: 에러 수집
- 빌드 로그에서 에러/경고 추출
- 에러 코드 기반 분류 (컴파일러, 빌드 시스템, 패키지 매니저, SDK)

#### Step 2: 근본 원인 분석
- 에러 코드별 패턴 매칭
- 의존성 충돌 탐지
- 환경 설정 문제 식별 (버전 불일치, 누락된 도구)
- 빌드 캐시/아티팩트 충돌 확인

#### Step 3: 수정안 제시
각 에러에 대해:
- 구체적 수정 코드/설정 제안
- 인프라 이슈 vs 코드 이슈 구분
- 우선순위 기반 수정 순서 제안

### Mode B: 빌드 성능 감사

#### Step 1: 빌드 구성 분석
- 빌드 설정 파일 파악
- 의존성 트리 깊이 확인
- 병렬 빌드 설정 상태

#### Step 2: 병목 식별
- 가장 느린 빌드 단계 식별
- 불필요한 재빌드 원인 탐지
- 린터/분석기 오버헤드 측정

#### Step 3: 최적화 권고
- 증분 빌드 개선 방안
- 캐싱 전략 제안
- 병렬화 기회 식별

## 빌드 안티패턴 (AP 코드)

| 코드 | 패턴 | 심각도 |
|------|------|--------|
| AP-01 | 절대 경로 하드코딩 | Critical |
| AP-02 | 빌드 출력 경로 충돌 | Critical |
| AP-03 | 순환 의존성 | Critical |
| AP-04 | 누락된 의존성 선언 | High |
| AP-05 | 불필요한 전체 재빌드 | Medium |
| AP-06 | 빌드 설정 조건 오류 | Medium |
| AP-07 | 미사용 의존성 | Low |

## 평가축 (3축)

| 축 | 평가 대상 | 등급 |
|----|----------|------|
| 근본 원인 정확도 | 에러 코드 기반 분류 정확성 | A/B/C/D |
| 수정안 실행 가능성 | 제안이 빌드 성공으로 이어지는 비율 | 1~5점 |
| 성능 추세 감지 | 빌드 시간 변화율 임계값(>10%) 감지 | % |

## 검증 체크리스트

- [ ] 모든 에러/경고 수집 완료
- [ ] 인프라 이슈 vs 코드 이슈 구분
- [ ] 각 에러에 구체적 수정안 포함
- [ ] 빌드 안티패턴 스캔 완료 (해당 시)
- [ ] 성능 병목 우선순위 정리

## 경계

- **하는 것**: 빌드 로그 분석, 설정 파일 점검, 최적화 권고
- **하지 않는 것**: CI/CD 파이프라인 구축, 빌드 직접 실행, 인프라 프로비저닝
