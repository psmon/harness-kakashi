# 조련사 (Tamer) Evaluation Axis

## Overview

조련사는 하네스의 전반적 품질과 활용도를 평가하는 3개 축을 사용한다.
평가 결과는 `harness/logs/tamer/` 에 MD 로그로 기록한다.

---

## Axis 1: 워크플로우 개선도 (Workflow Improvement)

기존 워크플로우 대비 효율성 변화를 평가한다.

**평가 항목**:
- 에이전트 간 협업 흐름의 명확성
- 중복 작업이나 불필요한 단계 제거 여부
- 병목 구간 식별 및 해소 여부
- 새로 추가된 워크플로우의 실용성

**등급**:
| 등급 | 기준 |
|------|------|
| A | 대폭 개선 — 명확한 효율성 향상, 병목 해소 |
| B | 소폭 개선 — 일부 개선점 발견, 실질적 효과 있음 |
| C | 변화 없음 — 구조적 변경 없음 |
| D | 악화 — 복잡성 증가, 불필요한 추상화 도입 |

---

## Axis 2: Claude 스킬 활용도 (Skill Utilization)

프로젝트 내 Claude 스킬들이 얼마나 잘 활용되고 연동되는지 평가한다.

**전제 조건**: 평가 전 반드시 다음을 읽는다:
- `.claude/skills/*/SKILL.md` (각 스킬 frontmatter) — 프로젝트 내 스킬 현황 파악

**평가 항목**:
- 스킬이 적절한 상황에서 호출/위임되고 있는가
- 하네스 에이전트가 관련 스킬을 참조하고 있는가
- 누락된 스킬 연동이 있는가
- 스킬 간 중복이나 충돌이 있는가

**점수**: 1~5점
| 점수 | 기준 |
|------|------|
| 5 | 완벽한 활용 — 모든 관련 스킬이 적재적소에 연동 |
| 4 | 우수 — 대부분 연동, 소수 누락 |
| 3 | 보통 — 일부 스킬만 연동, 개선 여지 있음 |
| 2 | 미흡 — 스킬 연동이 거의 없음 |
| 1 | 미활용 — 스킬 시스템을 전혀 활용하지 않음 |

---

## Axis 3: 하네스 성숙도 (Harness Maturity)

3-Layer 아키텍처의 충실도를 종합 평가한다.

**평가 항목**:
- Knowledge 레이어: 도메인 지식의 범위와 깊이
- Agents 레이어: 역할 정의의 명확성과 커버리지
- Engine 레이어: 워크플로우의 완성도
- 로그 축적량과 개선 피드백 루프 동작 여부

**레벨**:
| 레벨 | 명칭 | 기준 |
|------|------|------|
| L1 | 초기 | 빈 하네스, 디렉토리 구조만 존재 |
| L2 | 기본 | 조련사만 존재, knowledge/engine 비어 있음 |
| L3 | 성장 | 2~3개 에이전트 + 1개 이상 워크플로우 |
| L4 | 안정 | 도메인 지식 축적 + 다수 워크플로우 + 활발한 로그 |
| L5 | 성숙 | 자기 개선 루프 동작, 피드백 기반 지속적 하네스 진화 |

```markdown
- 등급: {A/B/C/D}
- 상세: {구체적 관찰사항}

## Axis 2: Claude 스킬 활용도
- 점수: {1~5}
- 상세: {스킬 연동 현황, 누락/개선 사항}

## Axis 3: 하네스 성숙도
- 레벨: {L1~L5}
- 상세: {각 레이어 현황}

## 종합 의견
{전반적 평가와 우선 개선 방향}

## 개선 제안
- {구체적 제안 1}
- {구체적 제안 2}
- {구체적 제안 3}
```

## 평가 빈도

- Mode A(Log & Eval) 동작 시마다 해당 에이전트의 평가축으로 자동 평가
- 사용자 명시 요청 시 ("평가로그를 점검해") 전체 3축 종합 평가
- 하네스 구조 변경(업그레이드) 후 성숙도 재평가
